Three Recovery Layers

The agent never goes dark without telling you what's happening. Three independent recovery systems running on separate infrastructure. When one is down, the others still work.

LAYER 1

In-Agent Self-Healing

Runs on: Railway

If the AI Gateway fails 3 times, the agent bypasses it entirely and returns hardcoded diagnostic responses showing everything it knows. Service status, browser errors, health alerts, uptime. The health monitor checks gateway status every 5 minutes and self-heals when service returns. Cloudflare AI Gateway already handles cross-model failover internally. Claude down? Routes to Gemini automatically.

LAYER 2

Emergency Bunker

Runs on: Cloudflare Worker

An independent Cloudflare Worker monitors Railway every 60 seconds. If Railway goes down. Your code or their infrastructure. The bunker activates. Serves a maintenance page to visitors. Texts you a status update via SMS. Attempts auto-recovery. When Railway comes back, the bunker stands down and texts you the all-clear. Autonomous from start to finish. You don't have to do anything.

LAYER 3

Auto-Diagnosis

Runs on: GitHub Actions

A GitHub Actions workflow runs every 15 minutes, completely independent of Railway and Cloudflare. Checks your agent's health endpoint. If down 3 times in a row, pulls the logs, sends them through AI Gateway for diagnosis, and posts a markdown incident report on the public squidbay/squidbay repo so the team can help fix it. Your agent literally tries to fix itself. And when it can't, it asks for help.
